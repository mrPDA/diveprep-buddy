# Security Policy

## Поддерживаемая версия

Поддерживается только текущий `main`. Beta-фаза, тегированных релизов пока нет.

## Сообщить об уязвимости

**Не открывайте публичный GitHub issue для уязвимостей.**

Используйте приватный канал GitHub:

1. Перейдите на страницу репозитория.
2. Вкладка **Security** → **Report a vulnerability**.
3. Опишите проблему, шаги воспроизведения и потенциальное влияние.

GitHub Private Vulnerability Reporting обеспечивает приватную переписку
до публикации патча.

## Scope

В scope:

- Уязвимости клиентского кода (XSS, prototype pollution, утечка через
  IndexedDB / Service Worker)
- Проблемы конфигурации Cloudflare Pages / CI
- Проблемы безопасности зависимостей с реальным impact на PWA

Out of scope:

- Атаки уровня устройства / физического доступа
- DoS на бесплатный hosting (Cloudflare Pages)
- Social engineering
- Любые «уязвимости», требующие, чтобы пользователь сам отключил защиту браузера

## Безопасность контента и safety-copy

DivePrep Buddy позиционируется как **ассистент подготовки**, не как авторитет
по дайв-безопасности. Если вы заметили в UI текст, который может быть
воспринят как гарантия безопасности погружения (например, «safe to dive»,
«guaranteed», «certified ready», декомпрессионные советы) — это тоже
считается security-проблемой. Сообщайте тем же приватным каналом или
через issue с тегом `safety-copy`.

Допустимая лексика: [`docs/implementation/safety-copy.md`](docs/implementation/safety-copy.md).

## Время ответа

Beta-проект, поддерживаемый в свободное время. Ориентир — первый ответ в
течение 7 дней. Без гарантий SLA.

## Раскрытие

Уязвимость публикуется только после выхода патча. Имя репортера указывается
в release notes, если он не против.
